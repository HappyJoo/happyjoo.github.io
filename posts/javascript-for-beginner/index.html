（有编程经验）JavaScript 入门避坑指南

  1. JavaScript 不区分整数和浮点数，统一用 Number 表示。
  2. 字符串是以单引号 ' 或双引号 " 括起来的任意文本
  3. 由于 JavaScript 的设计缺陷，不要使用 == 比较，始终坚持使用 === 比较。
  4. 另一个例外是 NaN 这个特殊的 Number 与所有其他值都不相等，包括它自己：唯一能判断 NaN 的方法是通过 isNaN() 函数：
  5. 由于精度问题，要比较两个浮点数是否相等，只能计算它们之差的绝对值，看是否小于某个阈值：Math.abs(1 / 3 - (1 - 2 / 3)) < 0.0000001; // true
  6. 大多数情况下，我们都应该用 null, undefined 仅仅在判断函数参数是否传递的情况下有用。
  7. JavaScript 的数组可以包括任意数据类型。出于代码的可读性考虑，强烈建议直接使用[]。
  8. 变量名是大小写英文、数字、$ 和 _ 的组合，不能数字开头，不能是关键字。申明用 var。
  9. 如果一个变量没有通过var申明就被使用，那么该变量就自动被申明为全局变量。
  10. ES6新增 var name = '小明';var age = 20;var message = `你好, ${name}, 你今年${age}岁了!`;
  11. 需要特别注意的是，字符串是不可变的，如果对字符串的某个索引赋值，不会有任何错误，但是，也没有任何效果。
  12. 请注意，如果通过索引赋值时，索引超过了范围，同样会引起Array，即数组大小的变化。
  13. 数组常用属性：length, indexOf, slice, push, pop, unshift, shift, sort, reverse, splice, concat, join
  14. 注意，最后一个键值对不需要在末尾加,，如果加了，有的浏览器（如低版本的IE）将报错。
  15. 如果属性名包含特殊字符，就必须用''括起来，访问这个属性也无法使用.操作符，必须用['xxx']来访问。
  16. 实际上 JavaScript 对象的所有属性都是字符串，不过属性对应的值可以是任意数据类型。
  17. JavaScript 规定，访问不存在的属性不报错，而是返回undefined。
  18. 由于 JavaScript 的对象是动态类型，你可以自由地给一个对象添加或删除属性。
  19. 如果我们要检测对象是否拥有某一属性，可以用 in 操作符。不过要小心，该属性可能是继承得到的。
  20. 要判断一个属性是否是该对象自身拥有的，而不是继承得到的，可以用hasOwnProperty()方法
  21. 在 if...else... 中，我们建议永远都要写上{}。
  22. JavaScript把null、undefined、0、NaN和空字符串''视为false，其他值一概视为true。
  23. 循环：for，for...in，while, do...while。
  24. Set和Map类似，也是一组 key 的集合，但不存储 value。所以，在Set中，没有重复的 key。用add()添加元素，用delete()删除元素。
  25. Map和Set是 ES6 标准新增的数据类型，请根据浏览器的支持情况决定是否要使用。
  26. 遍历用法：https://www.liaoxuefeng.com/wiki/1022910821149312/1023024358748480
  27. 函数就是最基本的一种代码抽象的方式。
  28. 定义函数：function name(variable) { do something }
